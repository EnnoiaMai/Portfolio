CSS Syntax

Each selector and statements within brackets define a rule.


    Selector {
        property1: value1;
        property2: value2;
    }
    Selector1, Selector2, ... {
        property: value;
    }

Selectors

Selector Code
ID Selector: one-of-a-kind thing, gives specific id to element to target. 

    #id {
        property: value;
    }
Class Selector: rules that apply to a group of elements, for mass changes. When applying multiple classes to an element, if the same properties are set, then the last class (most recent) in the css file will override that property. 

    .class {
        property: value;
    }
Tag Selector: selects all elements of a certain tag. 

    tag {
        property: value;
    }
Descendant Selectors: steps down the HTML tree and applies it to selectors within a specific path. 

    div li a {
        property: value;
    }
Child Selector: selects direct child of an element. 

    selector>childElement {
        property: value;
    }
Adjacent Sibling Selector: chooses only the element's next sibling. 

    selector+siblingElement {
        property: value;
    }
General Sibling Selector: chooses all siblings of selected element. 

    selector~siblingElement {
        property: value;
    }
Attribute Selector: chooses elements with the attribute. 

    selector[attribute] {
        property: value;
    }
Attribute with Value Selector: selects elements with attribute and specific value. 

    selector[attribute="value"] {
        property: value;
    }
Attribute with Beginning Value: selects element with attribute that begins with the value 

    selector[attribute^="value"] {
        property: value;
    }
Attribute with Ending Value: match any value that ends with a specific value 

    selector[attribute$="value"] {
        property: value;
    }
Attribute Containing Value: selects a value anywhere within attribute value 

    selector[attribute*="value"] {
        property: value;
    }
Wildcard Selector: selects every element. 

    * {
        property: value;
    }

If you are familiar with regular expressions, the use of ^ and $ are applied when filtering the values of the above attribute selectors.

Be careful with spaces when selecting elements

div .error - all elements under a div that is of class error

div.error - all divs that are of class error

div#navigation - the div with id "navigation" (unnecessary because id is unique anyway)

Pseudo Classes and Elements

Defines special state of an element. Gives you behavior in CSS without JavaScript.


    selector:pseudo-class {
        property: value;
    }
Pseudo Classes Definition
a:link Selects all unvisited links
a:visited Selects all visited links
a:hover (must come after a:link and a:visited) Selects links on mouse over
a:active (must come after a:hover) Selects the active link
:nth-child(n) For pattern selection, like every odd or even element or some multiple. Put arguments inside parentheses like keywords even or odd or some arbitrary number. E.g. 3n selects every 3rd element
:not(selector) Selects every element that is not the selector. Can exclude elements that you don't want to be affected by the rule.
:first-child Selects specified selector if it's the first child of its parent.
:last-child Selects every selected element that is the last element of its parent
:empty Selects every selected element that has no children
:checked Selects all checked <input element> elements
:valid Selects all <input element> elements with valid value
:invalid Selects all <input element> elements with invalid value
:required Selects <input element> elements with required attribute
:optional Selects <input element> elements without required attribute
:focus Selects <input element> that has focus
Pseudo Elements Definition
::after Insert content after every selected element
::before Insert content before every selected element
::first-letter Selects first letter of every selected textual element
::first-line Selects first line of every selected textual element
::selection Selects portion of an element selected by user

CSS Properties

Inheritance: properties are inherited from each child element's parents. child elements can override their parent's properties. Dimensional and layout properties are not inherited. The hierarchy of precendence is: id > class > tag

Property Value
color: #HEX | rgb | rgba | predefined;
background-color: #HEX | rgb | rgba | predefined; (Only applies to content and padding.)
background-repeat: norepeat | repeat-x | repeat-y;
background-image: url("path"); (Use relative path to make code portable. Can use external source to image but be cautious of their servers.)
overflow: visible | hidden | scroll | auto; (If content overflows an element's box. Works on block elements with a specified height.)
Text
Can use color to set color of text and background-color to set color of background.
text-align: center | left | right | justify; (Justify must be used with block level element.)
text-decoration: none | overline | underline | line-through;
text-indent: px;
text-transformation: uppercase | lowercase | capitalize;
letter-spacing: px; (The space between characters in a text.)
line-height: number | px | %; (The space between lines.)
word-spacing: px; (The space between words in a text.)
text-shadow: h-shadow(px) v-shadow(px) blur-radius(px) color;
Font
font-size: px | em | %
font-weight: bold | 100...900; (400 is normal, 700 is bold)
font-style: normal | italic | oblique;
font-family: "Times New Roman", Arial, Helvetica, sans-serif, ... (Fonts are used if available, if not the correctly applied property proceeds left-to-right.)
@font-face 

    @font-face {
        font-family: customFont;
        src: url();
    }

CSS Dimensions Layouts

Layouts are either liquid, fixed, or hybrid.


Responsive Design: design elements to look different on desktop and on mobile


Box Model of CSS: each element follows this model in which they contain their content and are surrounded by padding, then a border, then margins. Border and padding properties add to the dimensions of the element.


Dimensional and Layout Properties
Property Value
width: px | % | vw | auto;
height: px | % | vw | auto;
float: right | left | none;
margin(-top | -right | -bottom | -left): px | %;
padding(-top | -right | -bottom | -left): px | %;
Margin and Padding shortcut 

    margin: px; (applies to all)
    margin: px (top+bot) px (left+right);
    margin: px (top) px (left+right) px (bot);
Centering block elements 

    margin-left: auto;
    margin-right: auto;
border(-top | -right | -bottom | -left): px | %;
Border shortcut 

    border: px (width) style color;
border-radius: px; (Larger px means steeper, can apply border corners separately.) 

    border-top-left-radius: px;
border-image: url("") width(px) repeat; (Specifies image to use as border, border repeat as repeated, rounded or stretched.)

Positioning in CSS

  1. static - default, elements are within the flow of the document. Follows a top-down ordering,
  2. fixed - element is locked in place to the viewport and is out of the flow
  3. absolute - element positioned relative to first positioned(non-static) ancestor
  4. relative - element is positioned relative to where it would be normally

Values fixed, absolute, and relative positions take two values each from top/bottom and left/right in order to position elements.



    position: static | fixed | absolute | relative;

Shadow Effects

Cursors and Custom Cursors
cursor: auto | crosshair | help | wait | cell | progress | pointer | ...
cursor: url(""), url(""), ..., auto; (Defines list of custom cursors to use. Set auto at the end in the case that the custom cursor images can't be used.)

Display Property

Specifies type of box used for the element


    display: value;
Value Definition
inline Takes up as much space as element needs, will wrap.
block Fills to end of browser.
inline-block Like inline element with dimensional properties. Inside is block-level, element itself is inline-level.
flex A flexbox which prepares a single, invisible line where elements are aligned. Place elements on a row/column without needing to bother with floats or positioning of elements. Allows you to stack elements side-by-side. 

    <!-- HTML: -->
    <div class="flex-container">
        <div class="flex-item">Content<⁄div>
        <div class="flex-item">Content<⁄div>
        <div class="flex-item">Content<⁄div>
    <⁄div>
    

    /* CSS: */
    .flex-container {
        display: flex;
        width: 500px;
        height: 250px;
    }
    .flex-item {
        width: 100px;
        height: 100px;
        margin: 10px;
        background-color: blue;
    }

Creating a Simple Horizontal Menu

1) Create a list either ordered or unordered, within list create list elements and within list elements, create anchor tags.


    <!-- Html: -->
    <ul>
        <li><a href="">First Link<⁄a>
        <li><a href="">Second Link<⁄a>
        <li><a href="">Third Link<⁄a>
    <⁄ul>

2) Change display property to inline. Set min-width property so that menu doesn't fall apart when browser is resized or minimized.


    ul {
        text-align: center;
    }
    ul li {
        display: inline;
        list-style-type: none;
    }
    ul li a {
        color: #005;
        background-color: white;
        font-weight: bold;
        text-decoration: none;
        padding: 5px 10px 5px 10px;
    }
    ul li a:hover {
        color: white;
        background-color: #005;
    }

Creating a Simple Fixed Sidebar

1) Get rid of all margins and padding.


    * {
        margin: 0;
        padding: 0;
    }

2) Set height to 100% for html and body.


    html, body {
        height: 100%;
    }

3) Give sidebar a width, a height of 100%, fixed positioning, and make sure rest of body has margin-left equal to the width of the sidebar.


    <!-- Html: -->
    <div id="sidebar">
        Content for sidebar
    <⁄div>
    

    /* CSS: */
    #sidebar {
        width: 300px;
        height: 100%;
        position: fixed;
        top: 0px;
        left: 0px;
    }
    body {
        margin-left: 300px;
    }

Media Queries


    @media not|only mediatype and (media feature) {
        CSS code
    }
    @media screen and (min-width: 480px) {
        CSS code
    }

@media rule used to define different style rules for different media types/devices. The media feature or expressions resolve to true or false.


Can have different stylesheets based on the media


    <link rel="stylesheet" media="mediatype" and|not|only (media feature) href="mystylesheet.css">

Media Types
all For all media type devices
print For printers
screen For computer screens, tablets, smartphones etc.
Media Features
max-device-width max width of device like computer screen
max-device-height max height of device like computer screen
min-device-width min width of device like computer screen
min-device-height min height of device like computer screen
max-width max width of display area like browser window
max-height max height of display area like browser window
min-width min width of display area like browser window
min-height min height of display area like browser window
width the viewport width
height the viewport height
orientation orientation of viewport, landscape or portrait
