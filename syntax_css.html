CSS Syntax

Each selector and statements within brackets define a rule.

            Selector {
                property1: value1;
                property2: value2;
            }

Multiple Selectors:

Selector1, Selector2, ... { property: value; }

Selectors

ID Selector: one-of-a-kind thing, gives specific id to element to target

#idNAme {...}

Class Selector: rules that apply to a gruop of elements, for mass changes. When applying multiple classes to an element, if the same properties are set, then the last class will override that property.

.classNme {...}

Tag Selector: selects all elements of a certain tag

a {...}

Descendant Selectors: steps down the HTML tree and applies it to selectors within a specific path

li a {...}

Child Selector: selects direct child of an element

selector>childElement {...}

Adjacent Sibling Selector: chooses only the element's next sibling

selector+siblingElement {...}

General Sibling Selector: chooses all siblings of element

selector~siblingElement {...}

Attribute Selector: chooses elements with the attribute

selector[attribute] {....}

selector[attribute="value"] {....} selects element with attribute of a specific value

selector[attribute^="value"] {....} selects element with attribute that begins with the value

selector[attribute$="value"] {....} match any value that ends with a specific value

selector[attribute*="value"] {....} selects a value anywhere within attribute value

If you are familiar with regular expressions, the use of ^ and $ are conceptually similar

Wildcard Selector: selects every element

* {...}

div .error - all elements under a div that is of class error

div.error - all divs that are of class error

div#navigation - though unecessary because this is an id selector, the div that has the id "navigation"

Pseudoclasses

Gives you behavior in CSS without JavaScript

Examples Definition
a:link
a:visited
a:hover
a:active
:before
:after
:nth-child() For pattern selection, like every odd or even element or some multiple. Put arguments inside parentheses like keywords even or odd or some arbitrary number. E.g. 3n selects every 3rd element
:not() Can exclude elements that you don't want to be affected by the rule. Selector in parentheses must be present in the css.
:first-child Chooses the first child of the element. Other pseudo-selectors include :first-letter
:focus

For anchor tag pseudoclasses, want in this order: a:link{...}, a:visited{...}, a:hover{...}

CSS Properties

Inheritance: properties are inherited from each child element's parents. child elements can override their parent's properties. Dimensional and layout properties are not inherited. The hierarchy of precendence is: id -> class -> tag

color: #HEX | rgb | rgba | predefined;

background-color: #HEX | rgb | rgba | predefined; - only applies to content and padding

background-repeat: norepeat | repeat-x | repeat-y;

background-image: url("image"); - nest to use relative path to make code portable. Can use external source to image but be cautious of their servers.

overflow: visible | hidden | scroll | auto; - if content overflows an element's box, works on block elements with a specified height

Text

can use color to set color of text and background-color to set color of background

text-align: center | left | right | justify; - justify must be used with block level element

text-decoration: none | overline | underline | line-through;

text-indent: px;

text-transformation: uppercase | lowercase | capitalize;

letter-spacing: px; - the space between characters in a text

line-height: number | px | %; - the space between lines

word-spacing: px; - the space between words in a text

text-shadow: h-shadow(px) v-shadow(px) blur-radius(px) color;

Font

font-size: px | em | %

font-weight: bold | 100...900; - 400 is normal, 700 is bold

font-style: normal | italic | oblique;

font-family: "Times New Roman", Arial, Helvetica, sans-serif, ...

fonts are used if available, if not the property proceeds left-to-right

@font-face {
                font-family: customFont;
                src: url();
                }

CSS Layouts

Layouts are either liquid, fixed, or a hybrid of both

Responsive Design: design elements to look different on desktop and on mobile

Box Model of CSS

Each element follows this model wherein they contain their content and are surrounded by padding, then a border, then margins. Border and padding properties add to the dimensions of the element.

Dimensional and Layout Properties

margin: top | right | bottom | left;

padding: top | right | bottom | left;

shortcut can be margin: all; takes 1 value and sets all margins, margin: top+bot right+left; takes 2 values, margin: top left+right bot; takes 3 values. Applies to padding as well.

Note: center elements by setting left and right margins to auto

border: width(px) style color;

Individual properties include border-bottom | -top | -left | -right

border-radius: px; - larger px means steeper, can apply border corners separately e.g. border-top-left-radius: 10px;

border-image: url("") width(px) repeat - specifies image to use as border, border repeat as repeated, rounded or stretched

float: right | left | none;

Positioning in CSS

position: static | fixed | absolute | relative;

fixed, absolute, and relative positions take 2 values each from top/bottom and left/right in order to position elements.

  1. static - default, elements are within the flow of the document. Follows a top-down ordering,
  2. fixed - element is locked in place to the viewport and is out of the flow
  3. absolute - element positioned relative to first positioned(non-static) ancestor
  4. relative - element is positioned relative to where it would be normally

Special Effects

Gradients

background: linear-gradient(color1, color2) - default direction is top to bottom

Shadow Effects

text-shadow: h(px) v(px) blur(px) color;

box-shadow: h(px) v(px) blur(px) spread(px) color;

Cursor Property and Custom Cursor

cursor: auto | crosshair | help | wait | cell | progress | pointer | ...

cursor: url(""), url(""), auto; - defines list of custom cursors to use. Set auto at the end in the case that the custom cursor images can't be used.

Display Property

Specifies type of box used for the element

inline - takes up as much space as element needs, will wrap

block - fills to end of browser

inline-block - like inline element with dimensional properties. Inside is block-level, element itself is inline-level

flex - a flexbox gives you single row where elements are aligned on an invisible line. Place elements on a row without bothering with floats or positioning of elements. Allows you to stack elements side-by-side.

            HTML:
            <div class="flex-container">
                <div class="flex-item">Content<⁄div>
                <div class="flex-item">Content<⁄div>
                <div class="flex-item">Content<⁄div>
            <⁄div>
            CSS:
            .flex-container {
                display: flex;
                width: 500px;
                height: 250px;
            }
            .flex-item {
                width: 100px;
                height: 100px;
                margin: 10px;
                background-color: blue;
            }

Creating a Horizontal Menu

Create a list either ordered or unordered, within list create list elements and within list elements, create anchor tags. Change display property to inline.

Note: set min-width property so that menu doesn't fall apart when browser is resized

Creating a sidebar (simple way)

Get rid of all margins and padding.

            * {
                maring: 0;
                padding: 0;
            }

Set height to 100% for html and body.

            html, body {
                height: 100%;
            }

Give sidebar a width, a height of 100%, float it left, make sure rest of body has margin-left equal to the width of the sidebar.

            Html: <div id="sidebar">...<⁄div>
            CSS: #sidebar {
                width: 300px;
                float: left;
                height: 100%;
                }

Media Queries

@media rule used to define different style rules for different media types/devices

@media not|only mediatype and (media feature){...}

Can have different stylesheets based on the media

<link rel="stylesheet" media="mediatype" and|not|only (media feature) href="mystylesheet.css">

Media Types

  • all - for all media type devices
  • print - for printers
  • screen - for computer screens, tablets, smartphones etc.

Media Features

  • max-device-width: max width of device like computer screen
  • max-device-height: max height of device like computer screen
  • min-device-width
  • min-device-height
  • max-width: max width of display area like browser window
  • max-height: max height of display area like browser window
  • min-width
  • min-height
  • width: the viewport width
  • height: the viewport height
  • orientation: orientation of viewport, landscape or portrait

Example:

            @media screen and (min-width: 480px){
                selector {...}
            }
