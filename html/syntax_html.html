HTML

HTML (Hyptertext Markup Language): a markup language to create and add structure to web pages. A document containing text.

  • Syntax uses elements to mark up document.
  • Elements are defined with angle brackets <>.
  • Elements contain attributes which can each be set a value.

Document Object Model (DOM): tree-like structure but not binary, the structure of HTML wherein each element has an object associated with it which can be referenced in JavaScript. Elements can be selected in CSS.


index.html is the first page loaded on the website.


XML: a markup language used to communicate with different systems


utf-8allows encoding of ~1,000,000 characters in Unicode, not ASCII.


Doctype Declaration: tells browser to load most current version of HTML, not a tag. Has definition of what every tag means and how to treat it

Forms


<form action="someFile.php" method="GET | POST" name="" enctype="">

Creates a form, the following explains the attributes:

  • action: action to be performed when form is submitted, usually a path to webpage on server (a php file)
  • method: specifies GET or POST HTTP methods. Use GET for search or lookup, submitted form data will be visible. Use POST when changing data on the server, sends the form data to be processed.
  • name: for different forms, must have name for each input field or else data won't be sent.
  • enctype: for how submitted data will be encoded.

Form Attributes

value
Sets initial value of input.
readonly
Input is readonly and can't be set.
disabled
Input field is disabled and can't be submitted.
size
Size in characters for input field.
maxlength
Maximum allowed length for input.
min
The input value must be greater than or equal to the specified value.
max
The input value must be less than or equal to the specified value.
autocomplete
If form should autocomplete, browser completes input values based on previously entered values.
autofocus
Input field should get focused when page loads.
placeholder
Gives hint describing expected value, the lightened text will disappear when a value is inputted by user.
pattern
Takes a regular expression to check against input's value.
required
Indicates that field must be filled out, can't be empty, will set it as invalid.
