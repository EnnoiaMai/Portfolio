PHP

PHP: server-side scripting language designed primarily for web development. A loosely typed language

  • The client browser sends a request to the web server, which contains PHP files that are processed, and a new page is sent back as HTML.
  • Use to handle forms, file off emails, make calls to the database, etc.
  • Either find a web host with PHP and MySQL support or install your own web server and then install PHP and MySQL.
  • Market Share: PHP(~83%), ASP.net(~14%), Java(~2.5%)
  • File extension is .php
  • Scripts should be given 644 permissions. Executable permissions aren't necessary, only need read persmissions. This is because there is a module in Apache, the web server, that will handle PHP requests, it's reading files not executing them.

PHP Interpreter: runs constantly in Apache. When a request comes for a PHP page, the server spawns a thread and hands it off to the PHP interpreter. The server can deliver ~10k pages per second but will drop to hundreds if it has to start up the PHP interpreter every time.


XAMPP: a PHP development environment, download it, navigate to htdocs subdirectory.

Coercion

PHP interpreter tries to coerce variables to be the same type. It will try to coerce expressions as much as it can. Stops when finds some type that is different.



    // Example 1
    $x = 12;
    $y = 'foo';
    $x + $y;    // gives 12, foo changes to 0 because not valid integer
    $x * $y;    // gives 0
    $x / $y     // gives undefined (outputs nothing)

    // Example 2
    $x = 12;
    &y = '2foo';
    $x + $y;    // gives 14, tries to coerce $y to an integer

    // Example 3
    $x = 12;
    $y = 'foo2';
    $x + $y;    // gives 12, stops after checking 'f' in 'foo2'

    '9' == 9;       // true
    '09' == 9;      // true
    '9' === 9;      // false, checks type as well
    '9abcd' == 9;   // true
    '9abcd' === 9;  // false

The following are all false in conditions:
0 0.0 "" "0" "0.0"

'abc' coerced to a number is false, but as a string it's true.

Variables

Naming Variables

  • $ must be placed before every variable name.
  • Can't start a name with numbers, can't use hyphens or whitespace.
  • Underscores are allowed.
  • Put all in lowercase or separate words with underscore.
  • Don't have to define data type.

    <?php
            $variableName = value;
            $string_var = "This is a string";
            $integer_var = 5;
            echo $string_var;
        ?>

Variable Scope

Global Scope: variables declared outside a function, can only be accessed outside.

Local Scope: variables declared within a function.

global keyword: used to access a global variable from within a function.


Global variables are also stored by PHP in the array $GLOBALS[index] where index holds the name of the variable.


    <?php
            $x = 5;
            $y = 10;
            function myTest() {
                $GLOBALS['y'] = $GLOBALS['x'] + $GLOBALS['y'];
            }
        ?>

Static Scope: use static keyword to prevent a local variable from being deleted. The variable is still local to the function.


    <?php
            function myTest() {
                static $x = 0;
                echo $x;
                $x++
            }
            myTest()    // prints 0
            myTest()    // prints 1
        ?>

Variable Interpolation

Using double quotes invokes the interpreter whereas using single quotes does not.

  • In JS, there is no difference between single and double quotes unless there are quotes inside quotes, to which it must be distinguished.
  • Variable interpolation allows for dealing with key-value pairs in PHP.
  • Using single quotes is more efficient around constant strings, use it if interpolation isn't needed.

In the following example, the single quotes around the value for a MySQL field are required, so double quotes can be used around the entire statement 



    // No variables needed to be interpolated
    $x = 'Hello World';

    // Variables needed to be interpolated
    $y = "$x" . "welcome";

    // Interpolation to formulate sql statement
    $sql_statement = "SELECT name FROM runner WHERE name='John';";

    // Also works
    $sql_statement = "SELECT name FROM runner WHERE name='$_POST[fname]';";

Here and Now Documents

Here document: a language construct. Used in Perl when you have to print a webpage and you don't have embedded tags like in PHP.

  • In the example, anything can be put for ENDB but capitalizing it is the convention.
  • Normally, when printing, whitespace is ignored. However, Here document will actually preserve formatting of the source code. Makes it easy to send a large block of code.
  • Can include variables in the heredoc and it will perform variable interpolation.
  • If quotes are present in the heredoc, they will go.


    print <<<ENDB
        <html>
        <head>
        The value of x is $x    // will print whatever x is, not literal '$x'
        <input type="text" .../>
    ENDB;

Now document: similar to heredoc but there won't be variable interpolation.


    print <<<'ENDB'
        $x = 2;
        echo 'The value of x is $x';    // will display this string literally
        echo "The value of x is $x";    // will get variable interpolation
    ENDB;

Arrays

Use array() function

Types of arrays:

  1. Indexed arrays: arrays with a numeric index
  2. Associative arrays: arrays with named keys
  3. Multidimensional arrays: arrays containing one or more arrays

Indexed Arrays


    <?php
            // Creating an array with automatically assigned index
            $cars = array("Volve", "BMW", "Toyota");

            // Creating an array with manually assigned index
            $cars[0] = "Volvo";
            $cars[1] = "BMW";
            $cars[2] = "Toyota";

            // Return length of array
            count($cars);

            // Loop through Indexed array
            for ($x = 0; $x < count($cars); $x++) {
                statements;
            }
        ?>

Associative Arrays


    <?php
            // Create with array() function
            $age = array("Peter"=>"35", "Ben"=>"37", "Joe"=>"43");

            // Create manually
            $age['Peter'] = "35";
            $age['Ben'] = "37";
            $age['Joe'] = "43";

            // Using the named keys
            echo "Peter is " . $age['Peter'] . " years old.";

            // Loop through Associative array
            foreach ($age as $x => $x_value) {
                echo "Key=" . $x . ", Value=" . $x_value;
                echo "<br>";
            }
        ?>

PHP Forms

Form Handling

When processing forms after submission to the php file, grab the data by referring to the name attribute of the input element.

Both GET and POST creates an array which holds key/value pairs. Keys are the names of the form controls and values are the input data. Use superglobals $_GET and $_POST to access the data.

$_GET: an array of variables passed to the current script via URL parameters.

  • Information using GET is visible to others.
  • Limit of 2000 characters.
  • Can bookmark.

$_POST: an array of variables passed to the current script via HTTP POST method.

  • Information using POST is invisible to others.
  • No limits on amount of information.
  • Cannot bookmark.

    <?php

fopen(file, mode): opens a file in a certain mode. Also creates a file if it doesn't exist.

fread(file, bytes): reads from an open file. bytes is the maximum number of bytes to read.

fgets(file): reads single line from file.

fgetc(file): read a single character from file.

feof(file): checks if end-of-file (EOF) reached.

fclose(file): close an open file


    <?php
            $myfile = fopen("file.txt", "r") or die("Unable to open file!");
            echo fread($myfile, filesize("file.txt"));
            echo fgets($myfile);
            echo fgetc($myfile);

            while (!feof($myfile)) {
                echo fgets($myfile);
            }
            fclose($myfile);
        ?>

Mode Description
r Open file for read only. File pointer starts at the beginning.
w Open file for write only. Erases content of file or creates a new one if it doesn't exist. File pointer starts at the beginning.
a Open file for write only. Existing data is preserved. File pointer starts at the end of the file. Creates a new file if doesn't it exist. Like appending to a file.
x Creates a new file for write only. Returns FALSE and error if file already exists.
r+, w+, a+, x+ Same descriptions as above but allows read/write.

File Upload

Configure php.ini file to allow file uploads. Look for file_uploads directive and set to On.

file_uploads = On


    <!-- HTML -->
    <form action="upload.php" method="post" enctype="multipart/form-data">
        Select image to upload:
        <input type="file" name="fileToUpload" id="fileToUpload">
        <input type="submit" value="Upload Image" name="submit">
    <⁄form>

File upload requires method post and enctype as multipart/form-data


    <?php
            $target_dir = "uploads/";
            $target_file = $target_dir . basename($FILES["fileToUpload"]["name"]);
            $uploadOk = 1;

            // Get file extension
            $imageFileType = pathinfo($target_file, PATHINFO_EXTENSION);

            // Check if image is actual or fake
            if (isset($_POST["submit"])) {
                $check = getimagesize($_FILES["fileToUpload"]["tmp_name"]);
                if ($check !== false) {
                    echo "File is an image - " . $check["mime"] . ".";
                    $uploadOk = 1;
                } else {
                    echo "File is not an image.";
                    $uploadOk = 0;
                }
            }

            // Check if File Already Exists
            if (file_exists($target_file)) {
                echo "Sorry, file already exists.";
                $uploadOk = 0;
            }

            // Limit File Size
            if ($_FILES["fileToUpload"]["size"] > 1000000) {
                echo "Sorry, file is larger than 1MB";
                $uploadOk = 0;
            }

            // Limit File Type
            if ($imageFileType != "jpg" && $imageFileType != "png" && $imageFileType != "jpeg") {
                echo "Sorry, only JPG, JPEG, and PNG are allowed.";
                $uploadOk = 0;
            }

            // Check if $uploadOk is set to 0 by an error
            if ($uploadOk == 0) {
                echo "Sorry, your file wasn't uploaded.";
            } else {
                if (move_uploaded_file($_FILES["fileToUpload"]["tmp_name"], $target_file)) {
                    echo "The file " . basename($_FILES["fileToUpload"]["name"]) . " has been uploaded.";
                } else {
                    echo "Sorry, there was an error uploading your file."
                }
            }
        ?>

Cookies

Cookie: identifies a user. A small file that the server embeds on a user's computer. Each time the same computer requests a page with a browser, it will send the cookie too. Can create and retrieve cookie values with PHP.

Create Cookies

setcookie(name, value, expire, path, domain, secure, httponly);

  • only name parameter is required
  • Retrieve value of cookie user by using global variable $_COOKIE
  • / means cookie available in entire website
  • function must appear before the <html> tag
  • value of cookie is URLencoded when sending it and decoded when received


    <?php
            $cookie_name = "user";
            $cookie_value = "John Doe";
            // 86400 is a day
            setcookie($cookie_name, $cookie_value, time() + (86400 * 30), "/");
        ?>

Sessions

Session: way to store user information in variables to be used across multiple pages.

  • Unlike cookie, info. is not stored on user's computer.
  • Internet is stateless, so web server doesn't know who you are or what you do.
  • Lasts until user closes the browser.

session_start(): starts a PHP session

session_unset(): removes all global session variables

session_destroy(): destroys the session

print_r($_SESSION): shows all session variable values for a user session

Use global variable $_SESSION to set session variables. Change session variables by overwriting it.



    <?php
            session_start();
        ?>

Filters

Validating data: determining if the data is in proper form

Sanitizing data: removing any illegal characters from the data

Filters: validates and sanitizes external input. Access the filter extension by using filter_list() function.


filter_var(): validates and sanitizes data. Filters a single variable with a specified filter

  • takes the variable to check and the type of check to use.


    <?php
            // Sanitize a String
            $str = "<h1>Hello World!<⁄h1>";
            $newstr = filter_var($str, FILTER_SANITIZE_STRING);

            // Validate an Integer, the first check done because if $int set to 0, function will say it's invalid
            $int = 100;
            if (filter_var($int, FITLER_VALIDATE_INT) === 0 || !filter_var($int, FILTER_VALIDATE_INT) === false) {
            	echo ("Integer is valid.");
            } else {
            	echo("Integer is not valid.")
            }

            // Validate IP Address
            $ip = "127.0.0.1";

            if (!filter_var($ip, FILTER_VALIDATE_IP) === false) {
            	echo("$ip is a valid IP address.");
            } else {
            	echo("$ip is not a valid IP address");
            }

            // Sanitize and Validate Email Address
            $email = "john.doe@example.com";
            $email = filter_var($email, FILTER_SANITIZE_EMAIL);

            if (!filter_var($email, FILTER_VALIDATE_EMAIL) === false) {
            	echo("$email is a valid email address.");
            } else {
            	echo("$email is not a valid email address.");
            }
        ?>

Other forms of validation and sanitizing using filter_var() include urls, integers within a range, IPv6 address, a url with a querystring, and removing HTML tags and all characters with ASCII values > 127.

Exception Handling

PHP doesn't have exceptions that will cause a crash, so program will continue even if something has failed. The database stops it from crashing and has its own controls to check against duplicate entries. Exception is not a possibility because the client expects an error. Throwing an exception will result in the client timing out, usually 2 minutes.


When an exception is triggered:

  • The current code state is saved.
  • Code execution switches to the exception handler function.
  • The handler may resume execution, terminate, or continue the script in a different part of the code.

Try: function using exception should be in a try block

Throw: how to trigger an exception, must have at least one catch

Catch: retrieves an exception. Creates object containing exception information.

  • You can have multiple exceptions and catch blocks.
  • You can re-throw Exceptions by throwing in a catch block but wrapping the original try-catch block in another try-catch block.


    <?php
            function checkNum($number) {
                if ($number > 1) {
                	throw new Exception("Value must be 1 or below");
                }
                return true;
            }

            try {
            	checkNum(2);
            	echo 'Number is 1 or below.';
            } catch (customException $e) {
                echo 'CustomException: ' . $e->getMessage();
            } catch (Exception $e) {
                echo 'Message: ' . $e->getMessage();
            }
        ?>

  • The class must extend from the Exception class.


    <?php
            class CustomException extends Exception {
            	public function errorMessage() {
            		// error message
            		$errorMsg = 'Error on line ' .$this->getLine().' in '.this->getFile() .': <b> '.this->getMessage().'</b> is not a valid Email address';
            		return $errorMsg;
                }
            }
        ?>
