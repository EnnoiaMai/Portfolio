CSS Syntax

Each selector and statements within brackets define a rule.


Selector {
    property1: value1;
    property2: value2;
}
Selector1, Selector2, ... {
    property: value;
}

Selectors

ID Selector: one-of-a-kind thing, gives specific id to element to target.

#id {
    property: value;
}
Class Selector: rules that apply to a group of elements, for mass changes. When applying multiple classes to an element, if the same properties are set, then the last class (most recent) in the css file will override that property.

.class {
    property: value;
}
Tag Selector: selects all elements of a certain tag.

tag {
    property: value;
}
Descendant Selectors: steps down the HTML tree and applies it to selectors within a specific path.

div li a {
    property: value;
}
Child Selector: selects direct child of an element.

selector>childElement {
    property: value;
}
Adjacent Sibling Selector: chooses only the element's next sibling. More specifically, selects all elements placed immediately after the selector.

selector+siblingElement {
    property: value;
}
General Sibling Selector: chooses all siblings of selected element. More specifically, selects all elements preceded by the selector.

selector~siblingElement {
    property: value;
}
Attribute Selector: chooses elements with the attribute.

selector[attribute] {
    property: value;
}
Attribute with Value Selector: selects elements with attribute and specific value.

selector[attribute="value"] {
    property: value;
}
Attribute with Beginning Value: selects element with attribute that begins with the value

selector[attribute^="value"] {
    property: value;
}
Attribute with Ending Value: match any value that ends with a specific value

selector[attribute$="value"] {
    property: value;
}
Attribute Containing Value: selects a value anywhere within attribute value

selector[attribute*="value"] {
    property: value;
}
Wildcard Selector: selects every element.

* {
    property: value;
}

If you are familiar with regular expressions, the use of ^ and $ are applied when filtering the values of the above attribute selectors.

Be careful with spaces when selecting elements

div .error - all elements under a div that is of class error

div.error - all divs that are of class error

div#navigation - the div with id "navigation" (unnecessary because id is unique anyway)

Positioning in CSS

  1. static - default, elements are within the flow of the document. Follows a top-down ordering,
  2. fixed - element is locked in place to the viewport and is out of the flow
  3. absolute - element positioned relative to first positioned(non-static) ancestor
  4. relative - element is positioned relative to where it would be normally

Values fixed, absolute, and relative positions take two values each from top/bottom and left/right in order to position elements.



position: static | fixed | absolute | relative;

CSS Transitions

Transitions allow values of properties to change smoothly over time.

  • Must specify the CSS property to add an effect to and the duration.
  • To change several values, separate the individual transitions with commas.


/* General Syntax */
selector {
    transition: [property] [duration] [transition-timing-function] [delay];
}

/* Example */
div {
    width: 100px;
    height: 100px;
    background: red;
    transition: width 2s linear 1s;
}
div:hover {
    width: 300px;
}

/* Example of setting multiple transition effects */
selector {
    transition: background-color 0.5s linear 0s,
            width 1s ease 0.5s;
}

Flexbox

The source of the following information about Flexboxes came from this website


Flexbox layouts allow items to be distributed within a container, and is a dynamic way to let the child elements inside the container grow or shrink to fill up the space. It is a way to layout items dynamically.

  • The parent container must have its display property set to flex.

The following are properties associated with the parent container:


Property
Description
Value
flex-direction
Determines direction in which items will be laid out.
row | row-reverse | column | column-reverse
flex-wrap
Determines if items will be able to wrap around the container.
wrap | no-wrap | wrap-reverse
justify-content
Determines alignment of items along the main axis.
flex-start | flex-end | center | space-between | space-around
align-item
Determines alignment of items along the cross-axis.
flex-start | flex-end | center | baseline | stretch

The following are properties associated with the items within the container:


Property
Description
Value
order
Order in which the item is added to the container.
An integer. Default value is 0.
flex
Shorthand for flex-grow, flex-shrink, and flex-basis in that order.
Default value is 0 1 auto.
flex-grow
Allows item to grow to fill the remaining space.
Value is a proportion or a ratio. A value of 1 for this property for all items means that they will distribute space evenly.
flex-shrink
Determines if item will shrink.
Value is a proportion or a ratio. A value of 0 means that the item will not shrink.
flex-basis
Sets the size of the element. This size is applied before the space left over is distributed.
A length, like a percentage, a px, etc. Default value is auto.
align-self
Sets the alignment on the cross-axis of the individual item.
flex-start | flex-end | center | baseline | stretch | auto

Example of using Flexbox to Layout Contents

The following is an example of how to style a website to make a standard layout in which there is a horizontal menu that sticks at the top while scrolling down, content that is automatically sized in the middle, and a footer that sticks to the bottom of the page.



/* The body will be the outer container that holds the menu, the content, and the footer */
body {
    display: flex;
    flex-direction: column;
    height: 100vh;
}
/* Set the position to fixed so that the menu is out of the flow */
.menu {
    position: fixed;
    top: 0px;
    left: 0px;
    z-index: 10;
}
/* Content should grow to fill the remaining space, but shouldn't shrink */
.content {
    flex: 1 0 auto;
}
/* Footer should not shrink */
.footer {
    flex-shrink: 0;
}

Grid

The source of the following information about Grids came from this website


The Grid layout is a two-dimensional layout system that can handle both rows and columns. The page is divided into as many columns as specified by one of the grid properties. However, the max is 12 columns. Rows and columns can stretch or overlap in terms of the amount of space it takes up.

  • The parent container must have its display property set to grid.
  • The parent container is called the grid container, and the items within this grid are the grid items. There are grid lines which divide and distinguish each grid cell.

The following are properties associated with the grid container:


Property
Description
Value
grid-template-columns and grid-template-rows
Specifies number of columns and rows the grid will have. The lines can be named, and the size is a length such as a percentage, px, or a fraction (fr). If a custom name is not defined, then the line name is numerical.
size | name size
grid-template-areas
Lays out grid items by referencing the items by their grid-area property. If the grid-area name is repeated, then that item spans across those cells. The period means an empty cell.
grid-area name | . | none
grid-column-gap and grid-row-gap
Sets space between the columns and rows. More specifically, it is the width of the grid lines.
size
grid-gap
Shorthand for grid-row-gap and grid-column-gap. Takes size values.
size(grid-row-gap) size(grid-column-gap)
justify-items
Aligns grid item contents along the row axis. Applies to all grid items in the parent container.
start | end | center | stretch
align-items
Aligns grid item contents along the column axis. Applies to all grid items in the parent container.
start | end | center | stretch
justify-content
Aligns the grid within the grid container along the row axis. Only works if the size of the grid is smaller than the size of the grid container.
start | end | center | stretch | space=around | space-between
align-content
Aligns the grid within the grid container along the column axis.
start | end | center | stretch | space=around | space-between

The following are properties associated with the grid items:


Property
Description
Value
grid-column-start, grid-column-end, grid-row-start, grid-row-end
Sets position of the grid item in the container. Refers to the starting and ending grid lines and allows the grid item to span across multiple rows and/or columns.
number | name | span number | span name | auto
grid-area
Name of the grid item so that it could be referenced by the parent container when setting the grid-template-areas property.
name
justify-self
Aligns individual grid item's content along the row axis.
start | end | center | stretch
align-self
Aligns individual grid item's content along the column axis.
start | end | center | stretch

Example of using the Grid to Layout Contents

The following examples creates a grid with 3 columns but an undefined amount of rows, intentionally done to avoid having a fixed number of rows. The columnes are spaced out evenly by using the fraction (fr) unit.



.grid_container {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    grid-gap: 36px 24px;
}
.grid_container div {
    background-color: #FFFCF8;
    box-shadow: 0px 0px 1px 0px grey;
}

Creating a Simple Fixed Sidebar

1) Get rid of all margins and padding.


* {
    margin: 0;
    padding: 0;
}

2) Set height to 100% for html and body.


html, body {
    height: 100%;
}

3) Give sidebar a width, a height of 100%, fixed positioning, and make sure rest of body has margin-left equal to the width of the sidebar.


<!-- Html: -->
<div id="sidebar">
    Content for sidebar
<⁄div>


/* CSS: */
#sidebar {
    width: 300px;
    height: 100%;
    position: fixed;
    top: 0px;
    left: 0px;
}
body {
    margin-left: 300px;
}

Get input textbox to fill Parent Container

Include the box-sizing property with the value of border-box in the CSS for the parent container of the input element. The textbox will then stretch to fill its parent div in width.

Then, set the width of the input element to some percentage.


For example,


div {
    box-sizing: border-box;
    padding: 4px 16px;
    border: 1px solid grey;
}
input {
    width: 100%;
}
